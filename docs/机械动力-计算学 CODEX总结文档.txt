机械动力：计算学 / Create: Computation 项目总结
更新时间：2026-06-01

一、项目总定位

模组正式方向定为：

机械动力：计算学
英文名：Create: Computation

这是一个以“计算机发展史”为主线的 Create 附属模组。

核心目标不是只做晶圆、芯片或制导系统，而是让玩家亲手经历“计算”从原始记录到机械计算、可编程机械计算、电子计算、半导体制造，再到工业控制与制导应用的完整发展过程。

一句话概括：

玩家从计数石、算盘和齿轮进位开始，一步步发展出差分机、分析机、穿孔卡、继电器、真空管、晶体管、晶圆、集成电路和处理器，最终把成熟的计算能力接入 Create 工厂、飞行器和自动化系统。


二、两条主要分工线

当前项目可以分成两条互补路线：

1. 基础材料与计算科技树路线

协作方向：基础材料与计算科技树

核心问题：
计算从哪里来？

主要内容：
- 新矿物
- 基础材料
- 早期计数工具
- 机械计算结构
- 半导体制造
- 芯片与处理器
- 工业计算终端硬件

这条线负责模组的“科技树底座”。

2. 制导与控制应用路线

协作方向：制导与控制应用

核心问题：
计算拿来控制什么？

主要内容：
- 传感器
- Guidance Computer
- Computation Panel
- Program ROM
- PID / Clamp / Deadzone 等节点逻辑
- Actuator Interface
- Aeronautics 飞行控制
- 工厂自动化控制

这条线负责模组后期的“计算应用”。


三、两条路线的关系

两边不是冲突关系，而是上下游关系。

基础材料与计算科技树路线负责制造计算能力：

计数石 -> 机械算盘 -> 数位齿轮 -> 机械加法器 -> 差分机 -> 分析机 -> 穿孔卡 -> 继电器 -> 真空管 -> 晶体管 -> 晶圆 -> 集成电路 -> 处理器

制导与控制应用路线负责使用计算能力：

处理器 / 工业计算终端 -> 传感器输入 -> 节点程序 -> 控制输出 -> 飞行器 / 工厂 / 自动化机械

简化理解：

一条路线做“计算机本体和它的发展史”。
另一条路线做“计算机成熟以后能控制的系统”。

九、推荐近期开发顺序

当前最建议先补“前期计算”。

原因：
现在半导体内容已经有了，但如果直接从晶圆开始，科技树会像是突然跳到后期，缺少“计算学”的历史主线。

推荐下一批实现内容：

1. Counting Stone / 计数石
最早期的计数物品，用于手工记录。

2. Tally Board / 计数板
可放置的早期记录工具，显示简单计数。

3. Mechanical Abacus / 机械算盘
第一个可交互计算方块。

4. Digit Wheel / 数位齿轮
0-9 数位显示与机械转动的核心零件。

5. Carry Gearbox / 进位齿轮箱
实现从个位到十位的机械进位。

6. Mechanical Adder / 机械加法器
早期自动计算装置，后续差分机的基础。

7. Computation Strip / 计算纸带
早期输出结果载体，也可以作为后续穿孔卡和程序载体的前身。


十、两边接口建议

为了让你和朋友的内容自然接上，建议设定以下材料/物品依赖：

基础逻辑芯片 -> Guidance Computer
处理器 -> Advanced Guidance Core
Program ROM -> 穿孔卡 / 存储材料 / 芯片升级链
传感器外壳 -> 锡、钨、玻璃、陶瓷
Actuator Interface -> 黄铜构件、逻辑芯片、精密构件
工业计算终端 -> 后期控制系统总入口

这样你做的基础材料和计算主线，会成为朋友制导系统的前置条件。

十二、最终结论

项目总名：

机械动力：计算学
Create: Computation

你负责：
材料、矿物、早期计算、机械计算、半导体与芯片主线。

朋友负责：
传感器、制导计算、节点程序、控制输出、飞行器/工厂控制应用。

总体目标：
做一个既符合计算机发展史，又有机械动力风格的计算学附属模组。

最重要的原则：

不要把计算做成黑盒。
要让玩家看见数字如何被记录、齿轮如何进位、机械如何打印、程序如何被读取，最后计算能力如何被压缩进芯片并控制整个工业系统。
