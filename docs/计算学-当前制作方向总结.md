# 计算学当前制作方向总结

## 一句话方向

《机械动力：计算学 / Create: Computation》要做的不是另一套脱离 Create 的机械系统，而是建立在 Create 原有动线、应力、物流、红石和控制语言上的机械计算扩展。

## 核心原则

### 1. 先有动线，再有计算

任何计算结构都必须先回答：

1. 动力从哪里来
2. 应力如何沿动线传递
3. 哪些模块真正接在这条线上
4. 这条线最后完成了什么计算

不能再采用“附近扫描组件后中心方块自动工作”的黑盒逻辑。

### 2. 计算能力来自结构，不来自单个核心方块

禁止出现：

- 计算核心
- CPU 核心
- 万能电脑
- 单方块分析机
- 单方块差分机

计算能力必须来自完整的机械结构、模块数量、布局、节拍稳定性、同步情况和动线复杂度。

### 3. 模块必须靠实体接口连接

模块之间不能抽象共享能力，必须通过真实连接参与机器：

- 传动轴 / 齿轮 / 齿轮箱
- 节拍轴
- 数据总线
- 控制总线
- 纸带 / 打孔卡通路
- 红石接口

没有真正接上机器的模块，不算机器的一部分。

### 4. 玩家进行无代码编程

玩家通过下面这些手段配置机器：

- GUI 小范围参数配置
- 卡片 / 纸带
- 槽位
- 红石
- 方块朝向
- 机械连接关系

禁止文本脚本、命令行、Lua/JS/Python 式编程。

### 5. 优先复用 Create 原本就有的方块

能直接用 Create 原版方块完成的功能，不重复造轮子。

优先复用：

- 传动轴
- 大小齿轮
- 齿轮箱 / 垂直齿轮箱
- 离合器
- 换向器
- 序列齿轮变速器
- 皮带 / 链传动
- 漏斗 / 机械臂 / 传送带 / 仓储
- 红石链路

《计算学》新增的内容，只补计算专用模块。

## 差分机方向

差分机不是“打印机 + 附近几个组件”，而是机械数表机。

推荐第一版结构：

```text
动力输入
-> 稳速 / 节拍
-> 函数设定
-> 差分寄存器列
-> 累加机构
-> 数表 / 纸带输出
```

它的定位是：

**机械参数表工厂**

第一阶段可产出：

- 加工曲线表
- 校准表
- 轨迹表
- 弹道表

差分机的能力提升不应只是堆数量，而应体现在：

- 可处理阶数
- 精度
- 输出速度
- 节拍稳定度
- 同步难度
- 应力需求

## 分析机方向

分析机不是“机械打印机 + 组件加分”，而是早期机械程序控制机。

推荐结构：

```text
动力输入
-> 主节拍
-> 程序滚筒 / 读卡器
-> 寄存器
-> Mill 运算部
-> Store 存储列
-> 条件 / 顺序控制
-> 输出模块
```

它的定位是：

**早期可编程机械控制机**

第一阶段负责：

- 读取程序卡 / 参数卡
- 做顺序控制
- 做加法 / 比较 / 条件分流
- 输出校准卡、控制卡或机器配置结果

## 结构分层

以后设计计算结构时，尽量按这五层思考：

### 动力层

- 传动轴
- 齿轮
- 节拍
- 稳速
- 离合

### 表示层

- 数轮
- 寄存器
- 状态保存

### 运算层

- 累加
- 差分
- 加法
- 比较

### 控制层

- 程序滚筒
- 读卡器
- 条件分流
- 循环控制

### 传输层

- 计算轴
- 数据总线
- 控制总线
- 卡片 / 纸带通路

## 今后判断设计是否正确的标准

如果一个设计满足下面这些条件，就更接近正确方向：

- 先有动线，再有功能
- 模块必须真实接入机器
- 玩家能看懂输入、处理中、输出分别在哪
- 增加模块会改变结构能力，而不只是数值加成
- 机器有明确的输入输出方向
- 复杂功能需要更复杂的机械结构

如果一个设计出现下面这些情况，就说明方向跑偏：

- 中心方块扫描附近组件
- 单个方块完成全部计算
- 功能主要发生在 GUI 里
- 组件之间不需要实体连接
- 只是堆同一种模块提高数值

## 当前建议的制作顺序

1. 废掉现有“附近扫描式差分机 / 分析机”思路
2. 先建立主轴、节拍、模块接入规则
3. 先重做差分机第一版纵向切片
4. 再用同一套结构规则重做分析机
5. 最后扩展纸带、程序卡、总线和更复杂控制

## 当前最推荐的下一刀

**先把差分机重构成线性主轴式机械数表机。**

先把机器做对，再把功能做大。
