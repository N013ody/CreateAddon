# Create: Computation 设计提示词

你正在协助设计一个 Minecraft NeoForge 1.21.1 模组：《机械动力：计算学 / Create: Computation》。它是 Create 的附属模组，主题是让玩家从早期记录、机械计算、可编程机械计算、电子计算、半导体制造，一步步发展到工业控制与制导应用。

## 核心定位

这个模组不是普通芯片模组，也不是单纯的飞控/制导模组。它的主线是“计算能力如何被制造出来，并如何接入 Create 的机械工业系统”。

玩家应该能经历：

```text
计数与记录 -> 齿轮进位 -> 差分机 -> 分析机 -> 穿孔卡
-> 继电器/真空管/晶体管 -> 晶圆 -> 集成电路 -> 工业计算控制
```

## 最高设计原则

作为 Create 附属，本模组必须继承 Create 的高自定义和玩家搭建精神。

不要把核心玩法做成固定黑盒机器。差分机、分析机、计算机和工业控制器都应该由多个玩家可放置、可组合、可替换的机械模块组成。

每个模块都应贡献清晰、可量化的能力，例如：

- 计算速度
- 计算容量或上限
- 精度
- 存储量
- 程序长度
- 吞吐量
- 稳定性
- 校准质量

玩家应能通过“多建、少建、换组件、换布局、换动力输入”改变机器能力。

## 差分机方向

差分机不是普通计算器，而是“机械参数表工厂”。

可设计的模块包括：

- 函数设定鼓：决定函数或曲线类型。
- 差分寄存器柱：决定阶数、精度或可处理复杂度。
- 传动轴系：决定速度、同步性或稳定性。
- 数表打印机：输出加工曲线表、校准表、热处理表、光刻对准表等。

差分机输出不应停留在“收藏物品”，而要能被后续机器读取，影响半导体、加工、控制或制导系统。

## 分析机方向

分析机是早期可编程控制系统，不应是单方块机器。

可设计的模块包括：

- 穿孔卡读卡器：读取程序或参数输入。
- Mill 运算组件：提供计算速度。
- Store 存储柱：提供存储容量。
- 控制鼓或程序模块：决定程序长度和可用逻辑。
- 机械打印机：输出校准卡、控制纸带或机器配置。
- 动力/同步模块：影响速度、稳定性和错误率。

分析机的能力应由模块数量、布局、动力和连接方式共同决定。

## 半导体与芯片方向

半导体链路应该服务于“压缩计算能力”。

晶圆、芯片和处理器不是孤立材料，而是把大型机械计算结构逐步小型化、稳定化、自动化的结果。

机器设计应避免纯 GUI。优先使用：

- 世界内物品显示
- 旋转、抬升、压合、扫描、曝光等可见动画
- Create 风格的传动轴、齿轮、皮带和机械臂
- Ponder 教学说明

## 制导与控制方向

制导、传感器和控制系统是计算能力成熟后的应用章节。

它们应该读取传感器、程序卡、校准数据或计算终端输出，再驱动：

- 红石信号
- 机械离合器
- 舵面/推进器
- Create 工厂流程
- Create: Aeronautics 载具控制

不要把制导做成黑盒算法。玩家应该能看到输入、计算、判断和输出如何流动。

## Create 风格要求

每个新增方块都要回答：

- 它解决什么机械问题？
- 它提供什么可量化能力？
- 它如何接入动力或世界交互？
- 它如何被玩家组合进更大的结构？
- 它能否用 Ponder 在 30 秒内讲清楚？

每个玩家可见条目都必须有 Create 风格 tooltip。每个玩家可见方块都必须有 Ponder。

## 禁止方向

避免：

- 一个方块完成整台计算机。
- 只靠 GUI 选择程序或配方。
- 抽象数字面板替代可见机械结构。
- 只有固定配方、固定速度、固定产出的机器。
- 与 Create 原有机械语言脱节的电缆/RF/EU 能源系统。

## 推荐输出格式

设计新内容时，优先按以下结构回答：

```text
1. 方块/物品名称
2. 在科技树中的阶段
3. 它是不是模块，贡献什么能力
4. 如何接入动力或相邻结构
5. 玩家如何组合多个模块改变结果
6. 输入、处理、输出
7. 世界内视觉反馈
8. Shift tooltip 文案
9. W Ponder 场景
10. 最小可实现切片和验证方法
```

始终优先做一个可验证的纵向切片，再扩展下一层。
