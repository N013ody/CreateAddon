# 2026-06-04 计算方块悬停提示兜底

## 背景

数表打印机、机械打印机和计算组件已经写入了 `block.createaddon.*.tooltip.*` 语言键，也已经注册了 Ponder 场景，但在创造物品栏悬停时仍然只显示方块名，没有出现类似制导计算机和传感器的 `Shift` / `W` 提示。

## 判断

制导计算机等旧方块可以正常显示，说明客户端初始化和 Ponder 插件本身是有效的；计算方块的问题集中在方块物品的悬停说明入口没有稳定命中。为了避免继续依赖 Create 自动 Tooltip 链路的隐式匹配，本次给计算方块增加了专用的方块物品类作为兜底。

## 实现

- 新增 `ComputingBlockItem`，在物品悬停时主动显示：
  - 未按 Shift：`按住 [Shift] 可查看概要`
  - 始终显示：`按住 [W] 开始思索`
  - 按住 Shift：展开对应 `block.createaddon.xxx.tooltip.summary`、`conditionN`、`behaviourN`
- 将差分机和分析机相关方块物品注册改为 `registerComputingItem(...)`：
  - 函数设定鼓
  - 差分寄存器柱
  - 数表打印机
  - 穿孔卡读卡器
  - Mill 运算块
  - Store 存储柱
  - 机械打印机
- 保留 Create Tooltip/Ponder 注册，使已有的思索场景仍然挂在这些方块上。
- 补充中英文通用提示键：
  - `tooltip.createaddon.hold_shift`
  - `tooltip.createaddon.hold_w`
- 更新本地校验脚本，确保之后这些计算方块必须继续拥有悬停提示入口。

## 后续约定

以后新增计算学机器或组件时，如果属于差分机、分析机、机械计算模块等需要玩家理解搭建方式的方块，应优先使用 `registerComputingItem(...)`，并补齐 `summary`、`conditionN`、`behaviourN`。这样可以保证创造栏悬停时一定有使用说明，并且符合机械动力式的 `Shift` 查看概要、`W` 开始思索体验。
