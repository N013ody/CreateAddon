# 2026-06-04 计算机方块分面贴图标记制作记录

## 目标

把数表打印机和机械打印机的“每个面具体作用”从文字说明推进到方块外观上。

上一轮只补了 tooltip、Ponder 和文档说明，但游戏内方块模型仍然是 `cube_all`，六个面共用同一张贴图，所以玩家实际看到的外观没有变化。本次目标是让动力口、操作面和状态面在模型贴图上明显区分。

## 设计思路

数表打印机和机械打印机是轴向动力方块：

- 轴向两端负责接 Create 动力。
- 非轴向侧面负责玩家右击放入和取出。
- 另一个侧面保留为状态/计算窗口。

因此模型不能继续使用 `cube_all`。必须改成分面 `cube` 模型，并为不同面指定不同贴图。

## 实现方式

### 新增贴图标记脚本

新增：

- `.codex-tools/mark-computation-machine-faces.mjs`

这个脚本只负责数表打印机和机械打印机的分面贴图，不会运行旧的全量计算资源生成器，避免覆盖已经手工修好的语言、blockstate 和动力接入内容。

脚本生成：

```text
table_printer_end.png
table_printer_side.png
table_printer_status.png
mechanical_printer_end.png
mechanical_printer_side.png
mechanical_printer_status.png
```

### 模型改动

修改：

- `src/main/resources/assets/createaddon/models/block/table_printer.json`
- `src/main/resources/assets/createaddon/models/block/mechanical_printer.json`

从：

```text
minecraft:block/cube_all
```

改为：

```text
minecraft:block/cube
```

并指定：

- `up/down`：`*_end`，动力轴口面。
- `north/south/west`：`*_side`，操作面。
- `east`：`*_status`，状态/计算窗口面。

方块的 `axis=x/y/z` blockstate 仍然负责整体旋转，所以动力口会随放置轴向移动。

### 贴图含义

动力端面：

- 圆形轴口。
- 黄铜/深色机械口标记。

操作侧面：

- 纸张槽/取出口标记。
- 数表打印机使用纸张和曲线图案。
- 机械打印机使用绿色校准/输出标记。

状态面：

- 彩色状态窗口。
- 用于后续显示工作状态或过载提示的视觉预留。

### 验证脚本

修改：

- `.codex-tools/validate-computation-engines.mjs`

新增检查：

- 数表打印机和机械打印机模型必须使用分面 `cube`。
- 模型必须引用 `_end`、`_side`、`_status`。
- 对应贴图文件必须存在。

## 注意

Minecraft 不会热加载方块模型和贴图。要在游戏里看到这次改动，需要关闭当前客户端并重新运行测试客户端。

## 后续建议

下一步可以进入更精细的模型层：

- 用 Blockbench 给轴向端面做凸出的轴承口。
- 给操作面做纸张托盘、压印辊或取出口。
- 给状态面做可发光的红/绿状态灯。
- 在 Ponder 场景里用箭头分别指向动力口、操作面和组件扫描范围。
