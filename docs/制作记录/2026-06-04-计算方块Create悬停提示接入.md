# 2026-06-04 计算方块 Create 悬停提示接入制作记录

## 目标

让差分机和分析机相关方块像已有制导传感器一样，在物品悬停时显示 Create 风格提示：

```text
按住 [Shift] 可查看概要
按住 [W] 开始思索
```

之前虽然已经写了 `block.createaddon.*.tooltip.*` 语言键，也注册了 `TooltipModifier`，但创造栏里部分计算方块仍可能只显示物品名称。原因是 Create 的 `ItemDescription` 在物品悬停时会从物品本身解析 tooltip key，BlockItem 在某些情况下不会稳定落到对应的 `block.createaddon.*` key。

## 实现方式

修改：

- `src/main/java/io/github/n013ody/createaddon/client/CreateAddonClient.java`

新增：

```java
registerBlockTooltip(Item item, String blockDescriptionId)
```

这个方法会先调用：

```java
ItemDescription.useKey(item, () -> blockDescriptionId)
```

再注册：

```java
TooltipModifier.REGISTRY.register(item, new ItemDescription.Modifier(item, Palette.STANDARD_CREATE))
```

这样 BlockItem 的 tooltip 会明确使用对应方块的语言键，例如：

```text
block.createaddon.table_printer.tooltip.summary
block.createaddon.table_printer.tooltip.condition1
block.createaddon.table_printer.tooltip.behaviour1
```

## 本次接入对象

已显式绑定：

- `function_setting_drum`
- `difference_register_column`
- `table_printer`
- `punched_card_reader`
- `mill_arithmetic_block`
- `store_memory_column`
- `mechanical_printer`

尤其是：

- 数表打印机
- 机械打印机

现在应当像已有传感器方块一样，在创造栏悬停时出现 Create 的 Shift/W 提示。

## 验证脚本

修改：

- `.codex-tools/validate-create-tooltips.mjs`
- `.codex-tools/validate-computation-engines.mjs`

新增检查：

- `registerBlockTooltip(...)` 也算有效的 Create tooltip 注册。
- 数表打印机和机械打印机必须显式绑定到对应 block tooltip key。
