# 2026-06-03 差分机与分析机 MVP 制作记录

## 目标

为《机械动力：计算学 / Create: Computation》补上第一版差分机与分析机玩法闭环。

这次的核心原则是：计算机不是为了计算而计算，而是为了把计算结果转化成后续机器可使用的工业能力。

本次实现的闭环：

```text
差分机组件 -> 基础加工曲线表
分析机组件 -> 机器校准卡
机器校准卡 -> 后续接入晶圆设备/工业控制/制导系统
```

## 设计思路

### 差分机定位

差分机不是普通计算器，而是“参数表工厂”。

第一版用机械组件生成 `基础加工曲线表`，后续可以扩展成：

- 加工曲线表
- 光刻对准表
- 热处理曲线表
- 弹道校准表
- 航向修正表
- 转速补偿表

### 分析机定位

分析机不是高级差分机，而是早期可编程控制器。

第一版用 `基础加工曲线表` 作为输入，结合分析机组件输出 `机器校准卡`。后续可扩展为程序卡、控制纸带、ROM 前身、制导参数卡。

## 本次新增内容

### 差分机组件

- `function_setting_drum`：函数设定鼓
- `difference_register_column`：差分寄存器柱
- `table_printer`：数表打印机

使用方式：

```text
在数表打印机 4 格范围内放置：
- 至少 1 个函数设定鼓
- 至少 2 个差分寄存器柱

手持 空白数表纸 右击 数表打印机
等待加工完成
空手右击 数表打印机 取出 基础加工曲线表
```

### 分析机组件

- `punched_card_reader`：穿孔卡读卡器
- `mill_arithmetic_block`：Mill 运算磨坊
- `store_memory_column`：Store 存储柱
- `mechanical_printer`：机械打印机

使用方式：

```text
在机械打印机 4 格范围内放置：
- 至少 1 个穿孔卡读卡器
- 至少 1 个 Mill 运算磨坊
- 至少 1 个 Store 存储柱

手持 基础加工曲线表 右击 机械打印机
等待加工完成
空手右击 机械打印机 取出 机器校准卡
```

### 新增物品

- `blank_table_paper`：空白数表纸
- `basic_processing_curve_table`：基础加工曲线表
- `blank_punched_card`：空白穿孔卡
- `machine_calibration_card`：机器校准卡

## 实现方式

### 交互逻辑

新增通用计算装配方块类：

- `src/main/java/io/github/n013ody/createaddon/content/computation/ComputingAssemblyBlock.java`

这个类负责：

- 检查玩家手持输入物品是否正确
- 扫描 4 格范围内是否有足够组件
- 使用已有 `ChipMachineBlockEntity` 的计时加工逻辑
- 加工完成后允许空手右击取出产物

- `src/main/java/io/github/n013ody/createaddon/registry/ModCreativeTabs.java`

## 后续建议

下一步应该把 `machine_calibration_card` 接入已有半导体机器，例如晶圆切割机或晶圆抛光机。

推荐第一步接入方式：

```text
未使用机器校准卡：
- 只能进行基础加工
- 产出较低或速度较慢

使用机器校准卡后：
- 解锁稳定加工
- 解锁高级配方
- 为后续光刻/封装链路提供质量前置
```

这样可以让“计算”真正转化为机器能力，而不是停留在能生成表和卡片的阶段。
