# 2026-06-04 Create 附属技术学习基线

## 目标

系统学习当前工程对应版本的 Create 附属开发方式，避免以后继续凭旧版教程或印象制作方块。

## 已确认版本

- Minecraft 1.21.1
- NeoForge 21.1.219
- Create 6.0.10-280
- Flywheel 1.0.6
- Ponder 1.0.82

## 学习方式

- 阅读当前工程的 `build.gradle` 和 `gradle.properties`，确认依赖版本。
- 临时拉取 Create 官方 `mc1.21.1/dev` 分支源码作为同版本参考。
- 对照当前项目已有代码，重点检查：
  - `KineticBlockEntity`
  - `RotatedPillarKineticBlock`
  - `DirectionalKineticBlock`
  - `IRotate`
  - `IBE`
  - `BlockStressValues`
  - `ItemDescription`
  - `TooltipModifier`
  - Ponder 注册和场景写法
- 对照 Create 内部机器，例如 Millstone、Mixer、Press，整理速度驱动持续加工的写法。

## 关键结论

- Create 的 `getSpeed()` 会在过载时返回 0，因此机器逻辑应把 `getSpeed() == 0` 视为暂停或不可运行。
- 应力基础值按 1 RPM 记录，实际应力需求随转速增长。
- 轴向机器优先继承 `RotatedPillarKineticBlock`，并在资源里提供 `axis=x/y/z`。
- 附属模组不能直接使用 Create 内部 `CStress.setImpact(...)` 给非 Create 方块注册默认应力，因为源码中会检查 owner 是否为 Create。
- 本项目使用 DeferredRegister，不完全走 Create 自己的 Registrate tooltip 链路；计算学方块应继续使用 `ComputingBlockItem` / `registerComputingItem(...)` 作为稳定兜底。
- Ponder 和 Shift tooltip 是项目硬要求，不是可选装饰。
