# 2026-06-03 模块化计算设计原则记录

## 背景

项目负责人明确了《机械动力：计算学 / Create: Computation》的核心设计方向：

```text
计算学作为机械动力附属，必须以机械动力一贯的高自定义为前提。
```

这意味着差分机、分析机、计算机、半导体控制设备和后续制导控制系统，都不能只做成固定能力的单方块黑盒。

## 核心原则

以后设计和制作内容时，必须优先采用：

- 模块化结构。
- 玩家自主搭建。
- 可扩展计算能力。
- 可见机械逻辑。
- 世界内交互。

计算机器应该由多个组件共同组成，每个组件提供明确能力。例如：

- 计算速度。
- 计算上限或容量。
- 精度。
- 存储量。
- 程序长度。
- 吞吐量。
- 稳定性或校准质量。

## 分析机示例

分析机不应只是一个方块消耗物品并输出结果。

更符合方向的设计是：

```text
读卡器 + Mill 运算组件 + Store 存储柱 + 函数/控制模块 + 打印或输出模块 + 动力输入
```

这些组件的数量、种类、布局和动力条件，决定机器的计算速度、上限、可处理程序复杂度和输出质量。

## 差分机示例

差分机不应只是“放几个装饰组件后打印表”。

更合理的方向是：

```text
函数设定鼓决定函数类型
差分寄存器柱决定阶数/精度
传动和打印结构决定速度/吞吐
输出表质量影响后续机器校准
```

## 后续执行要求

以后继续制作差分机、分析机或其他计算设备时，先问：

```text
这个方块是一个黑盒机器，还是一个能被玩家组合进计算结构的模块？
它贡献了什么可量化能力？
玩家怎样通过多建、少建、换布局、换动力来改变结果？
```

如果回答不清楚，就先调整设计，再写代码。
