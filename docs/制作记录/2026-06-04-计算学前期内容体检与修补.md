# 2026-06-04 计算学前期内容体检与修补

## 目标

检查前面制作的计算学内容是否有需要修改的地方，重点看计算方块、tooltip/Ponder、转速与应力、方块实体清理和构建验证。

## 检查结论

整体结构是可继续往下做的：

- 计算打印机已经接入 Create 转速和应力。
- 数表打印机与机械打印机都有专用 block entity。
- 计算组件以模块化方式贡献计算单位。
- Shift tooltip 与 W/Ponder 兜底已通过验证脚本。
- 完整构建通过。

发现并修补的问题：

- 计算打印机和部分半导体机器内部保存输入/输出，但破坏方块时没有明确返还内部物品。
- 早期少数实现了 `IBE` 的方块没有统一调用 Create 的移除清理路径。

## 修改内容

新增内部物品返还：

- `ComputingMachineBlockEntity#dropContents(...)`
- `ChipMachineBlockEntity#dropContents(...)`

补齐方块移除清理：

- `ComputingAssemblyBlock#onRemove(...)`
- `ChipMachineBlock#onRemove(...)`
- `TestBlock#onRemove(...)`
- `GuidanceSensorBlock#onRemove(...)`
- `ThresholdControllerBlock#onRemove(...)`
- `RotaryFrameBlock#onRemove(...)`

验证脚本更新：

- `.codex-tools/validate-computation-engines.mjs`

以后会检查计算机器必须有破坏返还内部输入/输出的路径。

## 设计说明

破坏正在加工的机器时只返还输入，不掉落尚未完成的结果，避免玩家通过拆方块跳过加工时间。破坏已完成但未取出的机器时掉落输出，避免玩家丢失成品。

## 验证

```powershell
node .codex-tools\validate-create-tooltips.mjs
node .codex-tools\validate-computation-engines.mjs
.codex-tools\gradlew-dev.cmd build
git diff --check
```

结果：

- `Create tooltip coverage validated`
- `computation engine content validated`
- `BUILD SUCCESSFUL`
- `git diff --check` 无空白错误，只有已有换行格式提示。

## 后续建议

1. 给 `COMPUTING_MACHINE` 增加类似晶圆抛光机的简易渲染，让输入/输出和加工进度能被玩家看见。
2. 后续新增任何带内部物品的方块实体，都要同时考虑破坏返还、保存读取、同步显示和自动化接口。
3. 半导体线目前仍偏单方块流程，后续应逐步改成更 Create 风格的机械步骤和模块化扩展。
