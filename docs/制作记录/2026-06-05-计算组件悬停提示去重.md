# 2026-06-05 计算组件悬停提示去重

## 目标

修复差分机/分析机相关计算组件在创造物品栏悬停时，`按住 [Shift] 可查看概要` 与 `按住 [W] 开始思索` 被重复显示的问题。

## 现象与根因

游戏内已经能显示 Create 风格悬停说明，但计算学组件额外走了一条 `ComputingBlockItem.appendHoverText(...)` 的手写说明路径。

结果是同一批组件同时经过：

- `CreateAddonClient.registerBlockTooltip(...)`
- `ComputingBlockItem.appendHoverText(...)`

两条路径都会补上 Shift / W 提示，所以悬停文本出现双份。

## 设计判断

这里应当保留 Create 原生那条注册链，去掉手写兜底。

原因：

- 现在已经明确要求所有玩家可见方块统一接入 Create 风格说明
- 说明键已经统一落在 `block.createaddon.<id>.tooltip.*`
- 后续还要继续扩展模块化计算组件，悬停逻辑应保持单一来源，避免再次重复或互相打架

## 实现方法

1. 保留 `CreateAddonClient` 里的 `registerBlockTooltip(...)` 注册，作为计算组件说明的唯一来源
2. 精简 `ComputingBlockItem`，移除手写 `appendHoverText(...)`
3. 简化 `ModBlocks.registerComputingItem(...)`，不再传入未使用的 tooltip 参数
4. 更新校验脚本，要求计算组件必须显式注册 `registerBlockTooltip(...)`，并禁止 `ComputingBlockItem` 再手写悬停说明

## 变更文件

- `src/main/java/io/github/n013ody/createaddon/content/computation/ComputingBlockItem.java`
- `src/main/java/io/github/n013ody/createaddon/registry/ModBlocks.java`

- `.codex-tools/validate-create-tooltips.mjs`

- `node .codex-tools\validate-computation-engines.mjs`
  - 结果：失败，报错 `Computing block items must rely on Create tooltip registration instead of manually appending hover text`

修复后重新验证：

- `node .codex-tools\validate-computation-engines.mjs`
  - 结果：通过
- `node .codex-tools\validate-create-tooltips.mjs`
  - 结果：通过
- `.codex-tools\gradlew-dev.cmd build`
  - 结果：通过，`BUILD SUCCESSFUL`

## 后续建议

- 继续把所有新计算组件都维持为“Create 注册说明 + Ponder 教学”这一套
- 以后如果需要特殊悬停逻辑，优先扩展统一的 tooltip 注册链，不要再给单个组件手写第二套通用提示
