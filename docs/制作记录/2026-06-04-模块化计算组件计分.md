# 2026-06-04 模块化计算组件计分制作记录

## 目标

继续按《机械动力：计算学 / Create: Computation》的协作说明制作差分机和分析机内容。

本次目标不是新增更多黑箱机器，而是把已有差分机、分析机 MVP 从“满足最低组件数量即可运行”推进为“组件数量会提供可量化计算能力”。

## 设计思路

Create 附属的机械计算不应该是一块方块完成一切。玩家应该能通过多搭组件、调整布局、扩展结构来改变机器能力。

本次加入的第一版量化规则：

- 数表打印机仍需要最低差分机组件才能启动。
- 机械打印机仍需要最低分析机组件才能启动。
- 超过最低数量的组件会继续被扫描。
- 每个组件提供一定计算单位。
- 计算单位越高，加工时间越短。
- 加工时间有下限，避免无限堆叠后瞬间完成。

这让“更多齿轮和组件构成更强机器”的设计方向开始进入实际玩法，而不是只停留在文案。

## 实现方式

### 计算装配逻辑

修改：

- `src/main/java/io/github/n013ody/createaddon/content/computation/ComputingAssemblyBlock.java`

新增机制：

- 扫描 4 格范围内的所有所需组件。
- 计算当前组件总计算单位。
- 计算最低启动所需计算单位。
- 当当前计算单位高于最低值时，按比例缩短加工时间。
- 使用 `MIN_PROCESSING_TICKS` 保留最低处理时间。

公式方向：

```text
实际加工时间 = 基础加工时间 * 最低计算单位 / 当前计算单位
```

并使用下限保护：

```text
实际加工时间 >= 60 ticks
```

### 组件计分

修改：

- `src/main/java/io/github/n013ody/createaddon/registry/ModBlocks.java`

当前第一版组件权重：

差分机：

- 函数设定鼓：30 计算单位
- 差分寄存器柱：20 计算单位

分析机：

- 穿孔卡读卡器：25 计算单位
- Mill 运算磨坊：40 计算单位
- Store 存储柱：30 计算单位

这些数值是第一版玩法占位，后续可以继续拆分为速度、容量、精度、程序长度和稳定性等多个维度。

## 后续建议

下一步可以把“计算单位”继续拆成更像机械计算机的多维能力：

- Mill 提供计算速度。
- Store 提供容量或可处理表长度。
- 穿孔卡读卡器提供程序长度上限。
- 函数设定鼓提供函数复杂度。
- 差分寄存器柱提供差分阶数或精度。

再下一步可以让 `machine_calibration_card` 真正影响半导体机器，例如缩短晶圆抛光、降低失败率、解锁更高质量加工，形成：

```text
差分机 -> 数表
分析机 -> 校准卡
校准卡 -> 更强工业机器
```

这样“计算”会真正变成 Create 工厂里的可扩展能力。
