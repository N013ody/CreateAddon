# 2026-06-04 计算机转速与应力接入制作记录

## 目标

继续推进差分机与分析机的 Create 风格机械化。

本次目标：

- 数表打印机和机械打印机必须接入传动轴动力才能运行。
- 转速越高，计算运行越快。
- 转速越高，通过 Create 动力网络需要的应力也越多。
- 动力不足或应力过载时，机器不能正常推进计算。

## 设计思路

计算学作为 Create 附属，核心机器不应该只靠右击触发。差分机和分析机应该像 Create 机器一样吃动力、受转速影响、受应力容量约束。

本次采用的设计：

```text
组件数量 -> 计算单位
转速 -> 加工速度倍率
计算单位 + 当前加工状态 -> 应力影响
应力影响 * 转速倍率 -> Create 动力网络实际应力需求
```

也就是说：

- 多放组件可以提高计算单位。
- 提高转速可以缩短计算时间。
- 但高速运行会让 Create 网络消耗更多应力容量。
- 如果网络过载，计算会暂停推进。

这样玩家需要在“更大计算结构”和“更强动力系统”之间做工程取舍。

## 实现方式

### 新增动力计算方块实体

新增：

- `src/main/java/io/github/n013ody/createaddon/content/computation/ComputingMachineBlockEntity.java`

这个类继承 Create 的：

```text
KineticBlockEntity
```

负责：

- 存储输入物品、输出物品、加工剩余时间。
- 读取当前转速。
- 判断最低运行转速。
- 判断动力网络是否应力过载。
- 在加工开始和结束时刷新动力网络应力。
- 处理过程中如果动力不足或应力过载，则暂停倒计时。

最低运行转速：

```text
4 RPM
```

### 数表打印机与机械打印机接入动力轴

修改：

- `src/main/java/io/github/n013ody/createaddon/content/computation/ComputingAssemblyBlock.java`

原本计算装配方块继承普通半导体机器逻辑。现在改为继承：

```text
RotatedPillarKineticBlock
```

并实现：

```text
IBE<ComputingMachineBlockEntity>
```

现在它支持：

- 通过轴向连接传动轴。
- 启动时检查转速是否足够。
- 启动时检查动力网络是否已经过载。
- 根据组件计算单位和当前转速计算加工时间。
- 根据计算单位设置加工时的应力影响。

### 加工时间规则

基础方向：

```text
组件越多 -> 时间越短
转速越快 -> 时间越短
```

转速倍率以 32 RPM 作为基准，并设置最高 4 倍加速：

```text
speedFactor = clamp(speed / 32, 0.25, 4.0)
```

加工时间下限仍保留：

```text
60 ticks
```

避免玩家用极高转速和大量组件把早期机械计算压成瞬间完成。

### 应力规则

加工开始时会设置当前应力影响：

```text
stressImpact = 1.0 + computationUnits / 80.0
```

Create 动力网络会继续按实际转速把应力影响换算为真实应力需求。因此转速越高，网络需要的应力容量越多。

机器空闲时应力影响为 0。这样玩家可以保留大型计算结构，但只有真正计算时才产生主要动力负载。

## 后续建议

下一步可以继续把应力和计算能力拆得更细：

- Mill 决定速度倍率上限。
- Store 决定可处理数据容量。
- 穿孔卡读卡器决定程序长度。
- 函数设定鼓决定函数复杂度。
- 差分寄存器柱决定差分阶数和精度。

也可以加入更明确的失败/保护机制：

- 应力不足时计算暂停并保留输入。
- 过高转速可能降低稳定性或要求校准卡。
- 机器校准卡可以降低高转速计算的应力影响。

这样后续就能形成真正的机械工程玩法：玩家不是只合成一台计算机，而是在设计一套能承受速度和应力的计算结构。
