《机械动力：计算学》建模与贴图提示词

用途：
这份提示词用于制作 Minecraft 机械动力附属模组“机械动力：计算学 / Create: Computation”的机器方块模型与贴图。
目标不是做科幻黑盒，而是做“看得出工作原理的工业机器”：玩家一眼能看出输入、加工区、输出、动力结构和正在发生的制作过程。


一、完整主提示词

请为 Minecraft 1.21.1 NeoForge 模组“机械动力：计算学 / Create: Computation”设计一个 Create 风格机器方块模型与贴图。

这个模组主题是计算机发展史与工业计算流水线：从机械计算、差分机、分析机、穿孔卡、继电器、真空管、晶体管，到晶圆制造、集成电路和处理器。因此机器不能像普通魔法方块，也不能像随便堆起来的装饰块，而要像一台真实工业设备被 Minecraft 化、Create 化之后的结果。

建模顺序必须是：
1. 先理解这个方块在现实中对应什么机器，以及它在模组流水线中的功能。
2. 再设计轮廓，让玩家从 5 到 8 格外也能看懂它是什么。
3. 再加少量关键细节，不要先堆复杂花纹。
4. 最后重新设计贴图，并且必须以游戏内实际效果为准，不以单独渲染效果图为准。

整体风格要求：
- 主体以白色、浅灰、深灰为主，表现洁净、精密、计算设备和半导体设备的感觉。
- 可少量使用黄铜、铜色、深木色或黑色作为结构边框，但不能喧宾夺主。
- 可少量使用青色、蓝色或淡蓝色作为屏幕、晶圆、检测灯、能量/数据指示。
- 风格要接近机械动力 Create 的工业机械感，但不要直接复用 Create 的贴图。
- 机器应该简约、清晰、耐看，不要太花，不要满身花纹。

模型要求：
- 优先控制在 16x16x16 的普通方块范围内，除非机器功能明确需要突出结构。
- 先做低复杂度轮廓，保证远看能读懂；细节必须服务功能。
- 每台机器至少要有三个可读结构：
  1. 动力输入或传动结构：轴、齿轮箱、链条、侧面驱动盒、皮带轮等。
  2. 工作区域：加工台、晶圆盘、夹具、炉口、读卡槽、探针台等。
  3. 过程指示：窗口、灯条、进度槽、旋转盘、热光、检测指示等。
- 模型不要堆很多重复小方块，不要像杂乱的装饰雕塑。
- 避免多个 cuboid 重叠在同一平面导致贴图闪烁或 z-fighting。
- 避免巨大黑白靶心、过大的装饰面板、看不出用途的方块堆叠。

贴图要求：
- 所有贴图重新设计，不直接套用 Create 原版贴图。
- 像素风必须清晰，推荐 16x16 或 32x32，纹理要像 Minecraft 材质，不要像高清图片硬缩小。
- 主体贴图以白色陶瓷/白色金属外壳为主，边缘用浅灰和深灰做体积感。
- 细节数量要克制：一个面上只保留必要边框、螺丝、槽线、窗口或指示灯。
- 不要每个面都画满线条、蓝光、铜线、齿轮和装饰。
- 贴图在游戏内不能闪烁、不能错位、不能出现拉伸严重的面。
- 如果效果图好看但游戏里很丑，以游戏里为准重新改。

动画要求：
- 动画只做真实功能部件，不做无意义装饰。
- 晶圆抛光机：抛光盘或晶圆在加工时旋转，压头可轻微上下运动。
- 锯切机：锯片、线锯或切割头在加工时运动。
- 炉类机器：加工时显示热光、炉口亮度或风扇转动。
- 光刻台：加工时曝光头、镜头或蓝紫色光窗有状态变化。
- 测试台：加工时探针下压、指示灯扫描或测试窗口变化。
- 不加工时机器应该安静，只保留基础静态结构。

最终验收标准：
- 放进游戏后，从第一人称看，机器要比效果图更重要。
- 玩家应该一眼看出正面、输入区、加工区、输出区。
- 轮廓干净，不花，不乱，不像随机拼装。
- 贴图不闪，不糊，不拉伸，不重复到廉价。
- 和 Create 放在一起不违和，但有“计算学”自己的白色精密工业风格。


二、负面提示词

不要做成：
- 过于花哨、全身都是线条和发光纹路的机器。
- 复用机械动力 Create 原版贴图后简单拼接的机器。
- 像黑盒、普通箱子、魔法祭坛、科幻炮塔或随机装饰块。
- 大量铜色、木色、黑色堆满整个模型，导致主题不清楚。
- 巨大的黑白圆形靶心、过宽的装饰面板、无意义的蓝光条。
- 只有效果图好看，放进 Minecraft 后比例奇怪、贴图闪烁或看不清功能。
- 用很多小 cuboid 堆复杂度，但远看完全没有清晰轮廓。


三、单个机器设计模板

请按下面模板先写设计，再建模：

机器名称：
例如：晶圆抛光机 / Wafer Polisher

现实参考：
例如：CMP 化学机械抛光机，用旋转抛光盘、晶圆夹持头、压力控制和研磨液完成晶圆表面平坦化。

游戏功能：
例如：把 rough_wafer 加工成 polished_wafer，需要一段制作时间，而不是点一下瞬间完成。

视觉关键词：
白色精密机身、浅灰底座、深灰传动结构、青色晶圆窗口、简洁边框、少量黄铜/铜色机械连接。

必须可见的结构：
1. 中央圆形抛光盘或晶圆台。
2. 上方压头或夹具。
3. 侧面传动盒、轴、链条或齿轮箱。
4. 正面输入/输出槽或状态窗口。

动画点：
加工时中央盘旋转；压头轻微下压；状态窗口或晶圆有轻微亮度变化。

不要出现：
满身花纹、过大靶心、杂乱小方块、贴图闪烁、看不出加工区。


四、审查/返工提示词

请像美术总监一样审查这个 Minecraft 机器模型和贴图，只关注游戏内实际效果，不要被单独渲染图骗到。

请按下面标准给出问题和修改建议：
1. 远看轮廓是否清楚？5 到 8 格外能不能看出这是什么机器？
2. 是否能看出正面、输入、加工区、输出、动力输入？
3. 是否符合 Create 的工业机械语法，同时拥有“计算学”的白色精密设备风格？
4. 是否太花、太乱、太黑、太铜、太像随机拼装？
5. 贴图是否闪烁、错位、拉伸、噪点过多或颜色过脏？
6. 哪些细节应该删除？哪些功能部件应该保留或放大？
7. 如果只能改三处，优先改哪三处？

输出格式：
- 总体评价：一句话。
- 最大问题：列 3 条。
- 具体修改：列 5 条以内。
- 保留内容：列 3 条以内。
- 下一版目标：一句话。


五、给朋友的简短版

一句话要求：
做“白色精密工业风”的 Create 计算学机器，先让功能和轮廓读得懂，再加少量细节；不要花，不要复用 Create 贴图，不要只看效果图，必须以游戏内观感为准。

最重要的判断：
如果游戏里看起来像一堆装饰方块，就删细节、减颜色、放大核心功能部件，重新做清晰轮廓。
